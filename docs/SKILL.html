Linearizer

Human operator guide. Flattens any repository's AI-source into a single ordered, token-budgeted bundle with a table of contents — so a whole project (or one layer) can be fed to an AI in one prompt, or read top-to-bottom by a human. "Linearization" = serializing a tree/graph into one ordered document.
Status: built (self-test passing)  ·  Targets: Claude Code · Codex · Cursor  ·  Zero dependencies (Node ≥18)
Convention. This README.html is the human view. The AI-optimized source of truth is README.md — .md = AI optimization, .html = human view.

What it produces

One Markdown file (default LINEARIZED.md): a header, a table of contents (every file with a token estimate and included/omitted status), an omissions list when a budget drops files, then each file wrapped in ===== path ===== delimiters. A sample is in example-output/LINEARIZED.md.

read-only (writes 1 bundle) deterministic (same repo → identical bundle) no tokenizer / no network excludes .html + binaries

Run it

node tasks/skill-linearizer/scripts/linearize.mjs <repo> \
     [--out <file>] [--max-tokens N] \
     [--include <glob>] [--exclude <glob>] \
     [--order reading|graph|path]

node tasks/skill-linearizer/scripts/self-test.mjs   # offline determinism + behavior gate
  • Default <repo> is .; default output is LINEARIZED.md at the repo root — use --out /tmp/<name>.md to avoid overwriting in place.
  • --include / --exclude are repeatable globs, additive on top of the default policy.
  • The skill excludes its own install directory, so it never bundles itself.

Ordering modes

reading default

Entry contract (CLAUDE.md/AGENTS.md) + MANIFEST.json sequence first — entry docs, then docs they reference in document order, then manifest-named files — then the rest by path.

graph

Topologically orders markdown by cross-reference links: a doc comes after the docs it links to. Cycles broken deterministically.

path

Plain lexical path order.

budget --max-tokens

Includes files in order until the budget is hit, then stops and lists every omitted file. Never silently truncates.

Selection policy

Included by default: markdown/text/data (.md .mdc .txt .json .yaml .toml …) and common code (.ts .tsx .js .mjs .py .go .rs …). Always excluded: .html (the human view), images, fonts, media, archives, lockfiles, and any file with a NUL byte (binary). node_modules/, .git/, dist/, build output are never walked.

Determinism

  • Same repo state + options → byte-identical bundle (no clock/random in the body).
  • Every directory read, link resolution, and graph traversal sorts before iterating.
  • Token estimate = ceil(chars / 4) — documented, model-agnostic, reproducible.
  • Read-only apart from the single output file.
